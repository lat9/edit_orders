Usage Notes:

  1. A working copy of Zen Cart (v2.0.0 or later) running on a PHP version >= 8.0.2 is required by Edit Orders v5.0.0 and later.
  2. Some order-total modules are incompatibile with EO, especially if they require storefront conditions to perform their processing.
  3. Products added to an order via EO cannot include a "file-upload" option as that option-type is not currently supported.
  4. If your store's shipping-tax rate is different from the products' tax-rate, the setting My Store :: Show Split Tax Lines must be set to true to enable EO to correctly locate the shipping-tax rate on an order's initial edit.

Overview

Edit Orders (EO) is a Zen Cart admin module that allows administrative users to edit a customer's order to:

  • Change the customer, billing or shipping information
  • Add, delete, and edit products in an order
  • Update product quantities and pricing
  • Modify product attributes
  • Add, delete, and edit order totals

See Edit Order's Wiki for additional information.

Installing, Upgrading or Uninstalling

This section identifies the processes you'll use to initially install Edit Orders, upgrade or uninstall EO entirely.

Notes:

  1. Before you install EO v5.0.0 (or later), you'll need a working copy of Zen Cart 2.0.0 or later (2.1.0 recommended), running on a PHP version >= 8.0.2.
  2. Edit Orders provides an additional admin-sanitization script so that its inputs don't get over-sanitized by the zc155+ sanitizer. This, unfortunately, affects stores that have both EO installed and a cron-job set up to run the /admin/currency_cron.php script; see this Zen Cart issue for details.

Optional Core-File Overwrites

If your site uses either Coupons (ot_coupon) or Customer Group Pricing (ot_group_pricing), you have a higher probability of orders with "penny-off rounding" issues. That is, where an order's total value doesn't equal the sum of its individual order-total elements.

EO v4.7.0 (and later) provides optional updates to each of those order-total modules in an attempt to alleviate those situations. Please note:

  1. The change-basis for each module is that currently present in the master branch of Zen Cart's GitHub repository.
  2. There are no change-related comments in either of these modules.
  3. I have not tested these changes on a site that "displays prices with tax".

If you choose to install these overwrites, be sure to make a backup copy of the as-shipped Zen Cart version of each file. Do not simply rename a file, as that will lead to PHP errors on both your storefront and admin's Modules :: Order Totals tool! Choose a file extension that you'll remember, but don't use .php~ or .bak as those can lead to issues with PCI compliance.

Once the base files have been renamed, copy the updated versions from EO's /optional-core-overwrites sub-directory to the like-named sub-directory of your site's file-system:

  1. /includes/languages/english/modules/order_total/lang.ot_group_pricing.php (Added in v4.7.1)
  2. /includes/modules/order_total/ot_coupon.php
  3. /includes/modules/order_total/ot_group_pricing.php

Uninstalling

First, delete the files that were copied to your Zen Cart's file-system (restoring any core-file overwrites), then copy the contents of the plugin's /docs/edit_orders/uninstall/uninstall_eo.sql to your clipboard and paste into your admin's Tools->Install SQL Patches to complete the uninstall process.
